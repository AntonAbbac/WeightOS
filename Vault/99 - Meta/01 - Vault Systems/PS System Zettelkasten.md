O sistema PS ou *Problem Solver* no zettelkasten consiste em um método alternativo que eu mesmo desenvolvi junto com o zettelkasten padrão. É como se eu mesclasse o sistema original e complementasse com meu próprio método.

***
## 1. Fleeting IP

Imagine que há dois polos nessa pasta, um é o *fleeting* normal, ou seja, capturar ideias rápidas e objetivas — Deixando elas para serem estruturadas para outro momento disponível. Em contra partida o *IP* ou *Initial Problem*, é a sigla que significa *Problema Inicial*. É basicamente a mesma base, só que com conceito e finalidades diferentes. Eu capturo um problema, tomo uma hipótese inicial dele e deixo para estruturá-lo depois. Ela deve conter:

1. Causa do problema.
2. Opinião pessoal do problema e possíveis soluções rápidas ou superficiais.
3. Por que ele deve ser resolvido.

## 2. Literature Notes PS

Como o Fleeting *IP*, aqui é o mesmo sentido de dualidade. O *literature Notes* preserva o zettelkasten padrão, e o *PS* meu método complementar. No *Literature Notes* base, nós pegamos ideias de livros, artigos, vídeos ou outras fontes externas. No meu *PS* ou *Possible Solution*, é a sigla para *Solução Possível*, ou seja, extrair de fontes externas possíveis soluções para meu problema. Um clico de notas pode ser acabado aqui, sem chegar a ser uma *Permanent Note* ou *R*. Ela deve conter:

1. Ideias de outras pessoas com nomenclatura e domínio de fala(quotoes).
2. Minha opinião sobre as possíveis soluções e se elas são superiores, inferiores ou complementares as minhas próprias.
3. Links de soluções que completaram ou tentaram completar outro problema.

## 3. Permanent Notes R

No zettelkasten, este estágio de nota são as que ficam como notas importantes e atômicas — Quando essas notas estão prontas elas podem ser conectadas com outras formando uma rede de ideias próprias e de novas notas. O conceito *R* ou *Resolution*, que significa *Resolução*, é um complemento perfeito, pois foi feito, assim como os outros dois módulos *IP* e *PS*, ele desempenha o papel mestre aqui. Ele assim como o *Permanent Notes* ele é feito como a última nota do ciclo, mas com detalhes. Entre esses detalhes, ela carrega os seguintes elementos:

1. Deve conter reflexão pessoal e opinião sobre o problema
2. A solução deve ser feita de forma inteligente, tendo tanto mão externa inteligente, quanto interna com muita reflexão.
3. Links para outras notas de PS.

## Tags

No sistema Zettelkasten comum nós devemos usar aproximadamente 3 tipos de tags. Fleeting, Literature e Permanent.

No meu sistema modular complementar, o modo é o mesmo, porém a tag recebe um pequeno complemento de nome de acordo com a etapa. As tags são:

- Fleeting IP

- Literature PS

- Permanent R

Meu método complementar, o *PS System Kasten* é uma das ideias que casam com o método original sem comprometer estruturas ou alterar sua ideia de montar um sistema de conhecimento vasto dentro de uma estrutura de Vault(Obsdian). Os dois podem ser usados juntamente ao mesmo tempo sem problemas, o que faz uma jogada genial entre os dois mundos. Essa versão é apenas uma base para meu próprio método complementar e se preciso ele método pode ser atualizado ou transformado em um sistema próprio.
