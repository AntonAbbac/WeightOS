# 1. Core Principle of the Vault

This vault is **not organized by topics**, but by **information state and function**.

The defining question for where a note belongs is not

> "What is this about?"

but

> "At what stage of the system is this information?"

This principle prevents:

- topic-based folder sprawl
- duplicated notes
- difficulty retrieving actionable knowledge

This vault **requires regular processing**.

The structure is sound.

If needed, the next steps could be:

- formalizing movement rules between folders
- defining objective promotion criteria for notes
- designing a minimal daily/weekly workflow
- mapping common anti-patterns in this type of vault
